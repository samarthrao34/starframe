## Data Flow

### Commission Intake
1. User fills `commission.html` form → POST to `/api/commission`
2. Commission model stores in SQLite
3. Admin views/manages via `admin/` panel

### Payment Flow
1. Client initiates payment → `api/payment/create-order.js` creates Razorpay order
2. Razorpay checkout completes → `api/payment/verify-payment.js` verifies signature
3. Payment status updated in database

### Invoice Pipeline
1. Client requests invoice token → `api/invoice/token.js` issues JWT
2. Invoice HTML submitted → `api/invoice/send.js` generates PDF (html-pdf) and emails via Nodemailer

### Authentication
1. Login via username/password (JWT + session) or Google OAuth
2. Middleware validates JWT/session on protected routes
3. 2FA via Speakeasy (TOTP) — optional

### Real-time Analytics
1. Client loads `analytics-tracker.js` → establishes Socket.IO connection
2. Page visits, actions emitted to server
3. Server stores in `visitors_realtime` / `user_actions` tables
4. Admin dashboard queries aggregated analytics

## Conventions

**Naming:** PascalCase for models (Database, Commission), camelCase for routes and middleware
**Structure:** Flat route organization by domain (`admin`, `auth`, `analytics`, etc.)
**Auth pattern:** Middleware-first — auth/security middleware applied before route handlers
**Error handling:** Try-catch blocks with Winston logging, HTTP status codes returned
**Database:** Raw SQL queries, no ORM — all through singleton Database class
